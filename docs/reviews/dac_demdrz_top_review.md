## 3. Traceability Audit (REQ → MAS → RTL → test → TRM)

Audited all 42 IDs: REQ-001..029, PERF-001..005, CFG-001..003, ERR-001..004. **No broken links.**

- **PRD → MAS:** MAS §7 maps every ID to specific sections; spot-checked all 42 against MAS body text — every mapping is real (not just a table entry). All 10 PRD assumptions are restated in MAS §8 and bound.
- **MAS → RTL:** every specified mechanism located in the RTL and behaviorally identical to the MAS pseudocode: capture/phase equations (`dac_demdrz_top.v:110-117`, exactly MAS §4.1.1), fmt XOR before the capture register (line 125, D2), `SEED_*_EFF` localparams (84-86, CFG-002/D4), address decode with reserved `2'b11` (139-141, REQ-018), LFSR priority `rst > seed_ld > en` with zero substitution (`dac_lfsr16.v:49-61`, A4/A5), R-update rule (`dac_dem_coder.v:75-80`, REQ-004/010), 57-flop single-always output stage with pre-flop inversion (`dac_demdrz_top.v:191-213`, REQ-019/020/022/028/029). Reviewer independently re-derived the mod-7 reducer (all 15 digit-sum cases correct, incl. wrap arithmetic for 7..13 and the 14→0 case) and the rotator (true rotation, value-preserving; r=7 unreachable from mod7).
- **MAS corner cases → tests:** all 18 (C1–C18) mapped in test plan §3 and present in the TB as claimed; collision case C2 (write-wins, R from pre-write PRN, D8/M5) is explicitly checked in T-010 against a snapshot of the pre-write state.
- **REQ → test:** test plan §3 covers every ID with at least one passing test or a justified analysis-only disposition (REQ-020 structural part, PERF-005, ERR-004 — all argued correctly; the sim-provable parts of REQ-020/028 are covered by CHK-08). The checks are meaningful: cycle-exact MAS-derived reference model (TB mod-7 uses `% 7`, independent of the RTL digit-sum trick), independent weighted-sum/DRZ/complement/handshake invariants, hierarchical LFSR/R state locking, plus `dut2`/`dut3` parameter-override instances. Mutation results in §1 confirm fault detection.
- **REQ → TRM:** every requirement's behavior is documented in the TRM with its verifying test cited; port table (names, dirs, widths, reset values), parameter table, FSM table, latency/throughput tables, seed-protocol table, and reset table all re-checked line-by-line against the RTL — **mechanically consistent**, including the derived `data_req = (drz_en ? phase : 1'b1) & ~rst` equivalence and the `7'b0101010` reset value of `sw_msb_n`. Guide §2 example verified compilable (§1). Doc metadata (block/version/date/status/RTL hash) present and accurate on all five documents.
- **Standards (CLAUDE.md §6):** pure Verilog-2001 (no SV constructs — grep hits only the English word "logic" in comments); no `initial`/`#`/`$display` in `rtl/`; ANSI ports; all literals sized where width matters; no magic numbers (DRZ codes, address map, effective seeds are localparams); `<=` sequential / `=` combinational, never mixed; all `always @*` blocks fully assigned on every path (case+default; yosys `check -assert` confirms latch-free); single clock, sync active-high reset, every flop reset; FSM next-state logic separated and encoding documented. Sole deviation: `_q` vs `_r` naming (F-003, waived).
- **Spec-level:** no PRD↔MAS contradictions beyond F-007 (resolved). Behavior the spec left open is decided and *documented*, not silent: rotator r=7 (unreachable, documented), mid-stream `drz_en`/`dem_en` toggles (C8/C9), post-reset code-0 preamble (MAS §3, TRM §6, guide §3.3), combinational `data_req` (M6, TRM restriction 4). The mod-7 R distribution is ⌈64/7⌉-balanced (residue 0: 10/64 vs 9/64 others, ≈ +9% relative) — within REQ-010's "statistical bounds" acceptance and explicitly documented in MAS/TRM; not a finding.
